# Task and Target Assignment Enhancement Plan

## Overview
Enhance the task and target assignment system to provide fully functional assignment capabilities from the team page with real-time notifications for task assignment, completion, and overdue tasks.

## Goals
1. Enable task/target assignment from team member page
2. Implement real-time notifications for:
   - Task assignment to employees
   - Task completion notifications to admins
   - Overdue task notifications to admins
3. Maintain existing UI while enhancing functionality
4. Update backend to support enhanced assignment workflow

## Changes Required

### Frontend Changes

#### 1. Team Member Page (`frontend/src/pages/teammember.jsx`)
- Add assignment functionality to team member actions
- Create modal for assigning tasks/targets to selected team member
- Integrate with existing notification system for real-time updates

#### 2. Notification Context Enhancements
- Add new notification types for task assignment completion and overdue alerts
- Enhance `getNotificationIcon` and `getNotificationColor` functions

#### 3. Socket Connection
- Ensure proper handling of new notification event types

### Backend Changes

#### 1. Task Routes (`backend/routes/taskRoutes.js`)
- Enhance assignment workflow to trigger appropriate notifications
- Add overdue task checking mechanism
- Improve task completion notification logic

#### 2. Database Considerations
- Ensure proper indexing for notification queries
- Verify task assignment tables support enhanced tracking

## Implementation Details

### Notification Types to Add
- `task_assigned_to_employee` - When admin assigns task to employee
- `task_completed_by_employee` - When employee completes task (notify admin)
- `task_overdue_warning` - When task approaches deadline (notify admin)
- `task_overdue` - When task is past due date (notify admin)

### Frontend Implementation

#### Team Member Page Modifications
1. Add assignment button in team member action dropdown
2. Create assignment modal with:
   - Task/target selection dropdown
   - Due date picker
   - Priority selection
   - Notes field
3. Connect to backend assignment APIs
4. Trigger socket events for real-time updates

#### Notification Context Updates
1. Add new icon mappings for enhanced notification types
2. Add color coding for different notification priorities
3. Ensure backward compatibility with existing notification types

### Backend Implementation

#### Task Assignment Enhancement
1. Modify `/assign` endpoint to:
   - Send real-time notification to assigned employee via socket
   - Create appropriate notification record
   - Trigger admin notification for tracking

#### Task Completion Enhancement
1. Modify status update endpoint to:
   - Detect when task is marked as completed
   - Send notification to admin about completion
   - Update task achievements if applicable

#### Overdue Task Monitoring
1. Add background task or endpoint to:
   - Check for overdue tasks periodically
   - Send notifications to admins for overdue tasks
   - Send warning notifications as deadlines approach

## Integration Points

### Real-time Updates
- Use existing Socket.IO connection for instant notifications
- Leverage `join_notifications` and `join_admin` events for proper routing
- Ensure notification context properly handles new event types

### Data Flow
1. Admin assigns task via team member page
2. Backend processes assignment and emits socket event
3. Assigned employee receives real-time notification
4. Employee completes task via task management
5. Backend detects completion and notifies admin
6. System tracks overdue tasks and notifies admins
