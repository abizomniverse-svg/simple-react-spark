# Notification System Analysis

## Overview
The application has a comprehensive notification system with real-time updates, context-based state management, and multiple UI components for different notification types.

## Components

### 1. Notification Context (`frontend/src/context/NotificationContext.jsx`)
- **Purpose**: Global state management for notifications using React Context
- **Key Features**:
  - Manages `notifications` array (limited to 50 most recent)
  - Tracks `unreadCount` of notifications
  - Controls `showPanel` state for notification drawer
  - Socket.IO integration for real-time updates
  - Provides methods: `markAsRead`, `markAllAsRead`, `clearNotifications`
  - Helper functions: `getNotificationIcon`, `getNotificationColor`

### 2. Notification Bell Component (`frontend/src/components/NotificationBell.jsx`)
- **Purpose**: Main UI for notifications
- **Features**:
  - Bell icon with unread count badge
  - Sliding notification panel (appears from right)
  - Panel includes:
    - Header with unread count and action buttons
    - Notification list with type-based icons
    - Visual indicators for unread (blue dot)
    - Color-coded borders by notification type
    - Click-to-mark-as-read functionality

### 3. Navbar Notifications (`frontend/src/components/navbar.jsx`)
- **Purpose**: Task-specific notifications in top navbar
- **Features**:
  - Separate bell icon for task notifications
  - Fetches from `/api/task/notifications` endpoint
  - Shows unread count badge
  - Clicking navigates to task details page

### 4. Admin Notifications Page (`frontend/src/pages/adminnotifications.jsx`)
- **Purpose**: Dedicated admin notification management
- **Features**:
  - Shows system notifications (user approvals, etc.)
  - Displays profile change requests
  - Provides approve/reject actions for pending requests
  - Includes refresh mechanism for pending counts

### 5. Socket Connection (`frontend/src/socket/socket.js`)
- **Purpose**: Real-time communication backend
- **Features**:
  - Socket.IO client connection
  - Connects to backend URL from environment variables
  - Uses websocket transport with reconnection enabled

## Notification Color Coding
From NotificationContext:
- `missed_calls` or `task_not_completed`: Red (`border-l-red-500 bg-red-50`)
- `target_achieved` or `lead_converted`: Green (`border-l-green-500 bg-green-50`)
- `new_lead`, `proposal_created`, or `contract_created`: Blue (`border-l-blue-500 bg-blue-50`)
- `daily_task_summary`: Purple (`border-l-purple-500 bg-purple-50`)
- Default: Yellow (`border-l-yellow-500 bg-yellow-50`)

## Integration
- NotificationProvider wraps application (in index.js/main.jsx)
- NotificationBell used in navbar and dashboard layouts
- Socket events: `join_admin`, `join_notifications`, `new_notification`, `notification_read`, `mark_read`
- Admin notifications use REST API endpoints for approval workflows

## Current Status
The notification system appears to be fully functional with:
- Real-time updates via Socket.IO
- Proper state management through React Context
- Multiple UI access points (navbar bell, notification panel, admin page)
- Visual feedback for unread notifications
- Type-based icons and color coding
- Action capabilities (mark as read, clear, approve/reject for admins)
